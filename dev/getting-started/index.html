Getting started

This page walks through a first CodeEdit.jl edit from start to finish. The goal is not to show every feature, but to show the shape of the workflow: find a block, build an edit, review the plan, and apply it deliberately.

Installation

Install CodeEdit.jl from the Julia package manager. If the package is not yet registered, add it from its repository URL:

pkg> add https://github.com/perrutquist/CodeEdit.jl

Loading the package

julia> using CodeEdit

Creating a handle

CodeEdit.jl starts from source locations, but it does not make you edit individual line ranges. Use Handle to point at the block containing a location:

julia> h = Handle("examples/MyPackage.jl", 10)# examples/MyPackage.jl 7 - 11:
function foo(x)
    y = helper(x)
    z = y * 2
    return z
end

If line 10 is inside a function, h points to the whole function block, not just that line.

Display the handle to inspect the code:

julia> h# examples/MyPackage.jl 7 - 11:
function foo(x)
    y = helper(x)
    z = y * 2
    return z
end

or convert it to a string:

julia> source = string(h)"function foo(x)\n    y = helper(x)\n    z = y * 2\n    return z\nend\n"

Listing handles in files

List all blocks in one file:

julia> hs = handles("examples/MyPackage.jl")7 handles

   1 -  1: module MyPackage
   3 -  3: include("helpers.jl")
   5 -  5: const DEFAULT_LIMIT = 10
   7 - 11: function foo(x); y = helper(x); z = y * …
  13 - 15: function old_function_name(); return foo…
  17 - 17: end

List all blocks in files matching a glob:

julia> hs = handles("examples", "*.jl")9 handles
# examples/MyPackage.jl:
   1 -  1: module MyPackage
   3 -  3: include("helpers.jl")

  17 - 17: end
  EOF:

  1 - 1: helper(x) = x + 1
  EOF:

julia> hs = handles("examples/MyPackage.jl"; includes = true)9 handles

   1 -  1: module MyPackage
   3 -  3: include("helpers.jl")
   5 -  5: const DEFAULT_LIMIT = 10

  13 - 15: function old_function_name(); return foo…
  17 - 17: end

Searching text

Search within a set of handles:

julia> hs = handles("examples", "*.jl")9 handles
# examples/MyPackage.jl:
   1 -  1: module MyPackage
   3 -  3: include("helpers.jl")
   5 -  5: const DEFAULT_LIMIT = 10
   7 - 11: function foo(x); y = helper(x); z = y * …
  13 - 15: function old_function_name(); return foo…
  17 - 17: end
  EOF:

# examples/helpers.jl:
  1 - 1: helper(x) = x + 1
  EOF:
julia> matches = search(hs, "old_function_name")1 handle
# examples/MyPackage.jl:
  13 - 15: function old_function_name(); return foo…
julia> matches1 handle
# examples/MyPackage.jl:
  13 - 15: function old_function_name(); return foo…

The search result is a Set of handles, so you can inspect or edit each matching block.

Applying an edit with git

So far, we have only inspected source. To change it, create an edit value and choose how it should be applied.

For normal source code, use a git-backed version-control specification:

julia> repo = VersionControl("examples"; require_view=true)GitVersionControl("examples"; require_view=true)

With require_view=true, displaying the edit records the exact plan. When apply! runs, CodeEdit.jl plans the edit again and refuses to apply it if the plan has changed:

julia> h = only(search(hs, "old_function_name"))# examples/MyPackage.jl 13 - 15:
function old_function_name()
    return foo(1)
end
julia> edit = Replace(h, replace(string(h), "old_function_name" => "new_function_name"))Edit modifies examples/MyPackage.jl:
13c13
< function old_function_name()
---
> function new_function_name()
julia> apply!(repo, edit, "Rename old_function_name")[master beb2307] Rename old_function_name
 1 file changed, 1 insertion(+), 1 deletion(-)
Success

The edit was applied to disk and committed to git. This is the normal CodeEdit.jl workflow: source changes become small, named commits.

Inserting at the end of a file

Use eof_handle when inserting new code at the end of a file:

julia> h = eof_handle("examples/helpers.jl")# examples/helpers.jl EOF:
julia> edit = InsertBefore(h, raw"""
       
       another_helper(x) = helper(x) * 2
       """)Edit modifies examples/helpers.jl:
1c2,3
---
>
> another_helper(x) = helper(x) * 2
julia> apply!(repo, edit, "Add another helper")[master 6b05f9f] Add another helper
 1 file changed, 2 insertions(+)
Success

Applying without version control

For generated files, scratch files, or other workflows that should not create a commit, pass an explicit NoVersionControl specification:

julia> write("scratch.txt", "temporary = false\n")18
julia> h = Handle("scratch.txt", 1; parse_as=:text)# scratch.txt 1 - 1:
temporary = false
julia> edit = Replace(h, "temporary = true\n")Edit modifies ./scratch.txt:
1c1
< temporary = false
---
> temporary = true
julia> apply!(NoVersionControl(require_view=true), edit)Success
