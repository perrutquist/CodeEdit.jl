Editing code

Edits are represented as values that subtype AbstractEdit. Construct an edit, display it to review the diff, then apply it.

Displaying, printing, or stringifying an edit records the exact plan that was shown. apply! replans the edit and refuses to apply it if the new plan does not match the displayed plan. In REPL examples, leaving the semicolon off the edit = ... line displays the edit and marks it as displayed; calling display(edit) works too.

Use displayed! only when you intentionally want to mark an edit as reviewed without printing the diff.

Inserting code

Insert before a block:

julia> h = Handle("examples/insert-before.jl", 1)# examples/insert-before.jl 1 - 3:
function foo(x)
    x + 1
end
julia> edit = InsertBefore(h, raw"""
       const DEFAULT_LIMIT = 10
       
       """)Edit modifies examples/insert-before.jl:
0c1,2
---
> const DEFAULT_LIMIT = 10
>
julia> apply!(edit)Success.

Insert after a block:

julia> h = Handle("examples/insert-after.jl", 1)# examples/insert-after.jl 1 - 3:
function foo(x)
    x + 1
end
julia> edit = InsertAfter(h, raw"""
       
       function helper(x)
           return x + 1
       end
       """)Edit modifies examples/insert-after.jl:
3c4,7
---
>
> function helper(x)
>     return x + 1
> end
julia> apply!(edit)Success.

Use raw string literals such as raw"""...""" when writing Julia code as strings. They avoid accidental escaping of backslashes and dollar signs.

Combining edits

Use Combine, or the * shorthand, when multiple edits should be planned together:

julia> source = Handle("examples/source.jl", 1)# examples/source.jl 1 - 3:
function source()
    return :moved
end
julia> destination = eof_handle("examples/destination.jl")# examples/destination.jl EOF:
julia> edit = Combine(
           InsertBefore(destination, string(source)),
           Delete(source),
       )Edit modifies examples/destination.jl:
0c1,3
---
> function source()
>     return :moved
> end
Edit modifies examples/source.jl:
1,3c0
< function source()
<     return :moved
< end
---
julia> apply!(edit)Success.

Equivalent shorthand:

julia> source = Handle("examples/source-shorthand.jl", 1)# examples/source-shorthand.jl 1 - 3:
function source_shorthand()
    return :moved
end
julia> destination = eof_handle("examples/destination-shorthand.jl")# examples/destination-shorthand.jl EOF:
julia> edit = InsertBefore(destination, string(source)) * Delete(source)Edit modifies examples/destination-shorthand.jl:
0c1,3
---
> function source_shorthand()
>     return :moved
> end
Edit modifies examples/source-shorthand.jl:
1,3c0
< function source_shorthand()
<     return :moved
< end
---

Combined edits are validated after the full combined result is planned. This allows intermediate states to be temporarily invalid Julia syntax.

Planning and validation are all-or-nothing, but applying a combined edit that touches multiple files is best-effort at the filesystem level. If a later filesystem operation fails, earlier operations may already have been applied. Use version control so changes can be reviewed and recovered.

Applying edits without display

By default, apply! refuses to apply an edit until it has been displayed. This is intentional.

If you intentionally need to bypass display, use displayed!:

julia> h = Handle("examples/no-display.jl", 1)# examples/no-display.jl 1 - 1:
status() = :old
julia> edit = Replace(h, replace(string(h), ":old" => ":new"));
julia> displayed!(edit, true);
julia> apply!(edit)Success.

Use this with caution.
